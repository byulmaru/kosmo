## Context

`(tabs)` 레이아웃(`apps/web/src/routes/(tabs)/+layout.svelte`)은 현재 `lg:grid-cols-[20rem_1fr]` 2컬럼 그리드다. 좌측은 PROD-77에서 구현한 `SidebarNavigation`(데스크톱 고정 + 모바일 drawer 공용), 우측 트랙은 모바일 헤더·main·하단 탭 바를 묶은 콘텐츠 영역이다. Figma `05 Screens - Web`의 Home/1440 프레임은 WebSidebar 320 | CenterColumn 600 | RightRail 344(콘텐츠 320 + 좌측 여백 24) 구성을 확정했다.

## Goals / Non-Goals

**Goals:**

- 데스크톱(`lg` 이상)에서 좌측 내비 · 중앙 콘텐츠 · 우측 레일 3컬럼 그리드 셸을 도입한다.
- Figma 05 Web처럼 3컬럼 묶음을 가운데 정렬하고, 중앙 콘텐츠 폭을 최대 600px로 제한한다.
- 우측 컬럼은 후속 서브이슈(PROD-113)가 채울 수 있도록 빈 컨테이너로 자리만 확보한다.
- 기존 사이드바·모바일 drawer·하단 탭 바 동작을 그대로 유지한다.

**Non-Goals:**

- 우측 레일 위젯(ComposeWidget 등)·sticky 처리 — PROD-113
- `lg` 미만/이상 구간별 반응형 세부 정리(좁은 `lg` 구간의 컬럼 축소 대응 포함) — PROD-114
- 다단계 브레이크포인트(레일 숨김 → 내비 아이콘 축소) — PROD-118
- 사이드바 내부 디자인 정합 — PROD-127

## Decisions

- **우측 트랙 폭은 `minmax(290px,350px)` 가변폭(X 스타일)으로 한다.** Figma 1440 프레임 실측(344px 고정)도 후보였으나, 중간 폭 화면에서 중앙 컬럼을 덜 압박하도록 트위터/X의 가변 레일 수치를 채택했다(사용자 결정). Figma의 레일 콘텐츠 폭 320은 이 범위(290~350) 안에 있다.
- **중앙 트랙은 `minmax(0,600px)`로 잡고 3컬럼 묶음을 `justify-center`로 가운데 정렬한다.** Figma 1440 프레임(320|600|344 묶음을 중앙 정렬, 양옆 88px 여백)과 X의 중앙 정렬 컨테이너 구성을 따른다(사용자 결정). 중앙을 `1fr`로 두면 남는 폭을 전부 흡수해 전체 폭(full-bleed) 레이아웃이 되므로, 최대 600px로 제한하고 잔여 폭은 그리드 정렬로 양옆에 배분한다. `min 0`은 grid 기본 `min-width:auto`로 긴 콘텐츠가 레일을 밀어내는 것을 막는다.
- **우측 컬럼은 `hidden lg:block` 빈 `div`로 둔다.** `lg` 미만에서는 그리드 자체가 비활성이지만, 일반 플로우에서 빈 div가 영향을 주지 않도록 명시적으로 숨긴다. 컴포넌트 분리는 실제 위젯이 생기는 PROD-113에서 한다.
- **브레이크포인트는 기존 `lg` 단일을 유지한다.** PROD-114에서 같은 `lg` 기준으로 세부 조정하고, 다단계는 PROD-118로 이연한다.

## Risks / Trade-offs

- [`lg`(1024px)~1270px 구간에서 중앙 컬럼이 600px 미만으로 좁아짐 — 그리드 여유 폭 분배상 레일이 350px까지 함께 성장해 1024px에서 중앙은 약 354px] → 셸 골격 단계에서는 수용하고, 구간별 세부 조정은 PROD-114에서 처리한다. 콘텐츠 쪽은 고정 폭(`lg:w-[600px]`) 대신 트랙 폭을 따라가게 해(프로필·게시글 디테일 라우트) 레일 침범을 막는다.
- [빈 placeholder 컬럼이 데스크톱에서 시각적으로 비어 보임] → 의도된 중간 상태다. PROD-113이 같은 사이클에서 레일 컴포넌트를 채운다.
