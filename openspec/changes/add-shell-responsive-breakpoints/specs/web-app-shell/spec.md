## ADDED Requirements

### Requirement: Staged responsive breakpoints

웹 애플리케이션의 `(tabs)` 셸은 화면 폭에 따라 4단계로 레이아웃을 단계적으로 변형해야 한다(MUST). 단계 경계는 Tailwind 기본 브레이크포인트 `md`(768px)·`lg`(1024px)·`xl`(1280px)을 재사용하며, 커스텀 브레이크포인트 토큰을 추가하지 않는다(MUST). 모바일 ↔ 데스크톱 셸 경계는 `md`이며(MUST), `md` 이상에서는 (접힌) 사이드바가 항상 보이고 하단 탭 바·drawer는 표시하지 않는다(MUST). 좌측 사이드바는 `xl` 미만에서 아이콘 전용 레일로, `xl` 이상에서 풀 사이드바로 표시해야 한다(MUST). 각 단계에서 내비게이션 진입(드로어·아이콘 레일·풀 사이드바·하단 탭)과 글쓰기 진입이 동작해야 한다(MUST).

#### Scenario: 모바일 단계 (md 미만)

- **WHEN** 사용자가 `md` 미만 너비에서 `(tabs)` 페이지를 본다
- **THEN** 시스템은 상단 메뉴 헤더와 하단 탭 바, 전체 폭 콘텐츠를 표시한다
- **AND** 사이드바는 drawer로만 접근하며 우측 레일은 표시하지 않는다

#### Scenario: 아이콘 레일 + 피드 단계 (md~lg)

- **WHEN** 사용자가 `md` 이상 `lg` 미만 너비에서 본다
- **THEN** 시스템은 좌측 아이콘 레일과 중앙 콘텐츠 2컬럼을 표시하고, 우측 레일과 하단 탭 바는 표시하지 않는다
- **AND** 아이콘 레일에서 내비게이션·글쓰기·프로필 스위처가 동작한다

#### Scenario: 아이콘 레일 + 피드 + 우측 레일 단계 (lg~xl)

- **WHEN** 사용자가 `lg` 이상 `xl` 미만 너비에서 본다
- **THEN** 시스템은 좌측 아이콘 레일·중앙 콘텐츠·우측 레일의 3컬럼을 표시한다

#### Scenario: 풀 3분할 단계 (xl 이상)

- **WHEN** 사용자가 `xl` 이상 너비에서 본다
- **THEN** 시스템은 좌측 풀 사이드바(프로필 헤더 + 라벨)·중앙 콘텐츠·우측 레일의 3컬럼을 표시한다

#### Scenario: 데스크톱 글쓰기 진입

- **WHEN** 사용자가 `md` 이상 너비에서 글쓰기를 시작하려 한다
- **THEN** 시스템은 사이드바(아이콘 레일/풀 사이드바)의 글쓰기 버튼으로 `/compose` 진입을 제공한다

#### Scenario: 접힌 사이드바에서 프로필 스위처

- **WHEN** 사용자가 `md` 이상 `xl` 미만 너비에서 아이콘 레일의 프로필 아바타를 누른다
- **THEN** 시스템은 drawer를 열지 않고 프로필 스위처 드롭다운을 아바타 옆에 표시한다
