## Context

`add-main-three-column-shell`(PROD-112, 아카이브 완료)이 `(tabs)` 셸을 `lg` 이상에서 3컬럼으로 만들었고, 반응형은 `lg` 단일 경계만 둔다. PROD-118은 그 위에 트위터식 압축형 다단계 브레이크포인트를 얹어, 풀 3분할을 `lg`로 앞당기고 중간에 아이콘 레일 단계를 둔다.

## Decisions

- **Tailwind 기본 `md`/`lg` 재사용(압축형).** `md`(768)에서 아이콘 레일(80)+중앙, `lg`(1024)에서 풀 사이드바(320)+중앙(`minmax(0,600px)`)+우측 레일(290~350)로 전환한다. `lg` 시작점에서 중앙은 ≈414px로 수축됐다가 폭이 늘면 600까지 자란다. 풀 3분할 등장을 `xl`→`lg`로 앞당겨 좁은 데스크톱에서 더 촘촘하게 압축한다. 커스텀 `--breakpoint-*` 토큰은 쓰지 않는다.
- **모바일 경계 `lg`→`md`.** `md` 이상에서는 (접힌/펼친) 사이드바가 항상 보이고 하단 탭 바·drawer는 숨긴다. 아이콘 레일 단계는 `md`~`lg`, 풀 사이드바 3분할은 `lg` 이상이다.
- **아이콘 레일은 `SidebarNavigation` 내부 두 분기 + `lg` 가시성으로 전환.** 컴포넌트가 절대 위치·고정 폭으로 짜여 CSS만으로 5rem 레일로 접히지 않으므로, 풀 분기(기존 마크업, `hidden lg:flex`)와 아이콘 레일 분기(`lg:hidden`)를 한 컴포넌트에서 렌더해 스크립트·뮤테이션·스위처 상태를 공유한다. drawer surface는 항상 풀 분기를 쓴다. JS 미디어쿼리 없이 SSR-안전하다.
- **글쓰기 진입은 우측 레일 없는 단계만.** 아이콘 레일 단계(`md`~`lg`)엔 우측 컴포저·하단 탭이 없어 아이콘 레일에 `/compose` 글쓰기 버튼을 둔다. `lg` 이상은 우측 레일 컴포저가 담당하므로 풀 사이드바 글쓰기 버튼은 데스크톱에서 숨기고 drawer surface에서만 남긴다.
- **프로필 스위처 드롭다운**은 snippet으로 추출해 풀(아바타 아래)·아이콘 레일(아바타 우측 popover) 양쪽에서 위치만 바꿔 재사용한다.

## Risks / Trade-offs

- 데스크톱 surface에서 두 분기가 모두 DOM에 존재한다(한쪽은 `display:none`). 폼 `id` 충돌을 막으려 snippet에 surface별 suffix를 둔다.
- 기존 풀 분기의 raw hex 색상은 이번 범위에서 유지한다(후속 토큰 마이그레이션). 신규 마크업만 디자인 토큰을 쓴다.
