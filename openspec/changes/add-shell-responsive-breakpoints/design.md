## Context

`add-main-three-column-shell`(PROD-112)이 `(tabs)` 셸을 `lg` 이상에서 3컬럼으로 만들었고, 반응형은 `lg` 단일 경계만 둔다. PROD-118은 그 위에 트위터식 다단계 브레이크포인트를 얹는다.

## Decisions

- **Tailwind 기본 `md`/`lg`/`xl` 재사용.** 컬럼 폭 산수(풀 사이드바 320 / 아이콘 레일 80, 중앙 600, 우측 290)를 더하면 단계 경계가 768/1024/1280에 자연스럽게 떨어져, 커스텀 `--breakpoint-*` 토큰 없이 기본값으로 표현된다.
- **모바일 경계 `lg`→`md`.** `md` 이상에서는 (접힌) 사이드바가 항상 보이고 하단 탭 바·drawer는 숨긴다. Figma 1024 프레임(접힌 아이콘 메뉴)이 `lg`~`xl` 단계와 정합한다.
- **아이콘 레일은 `SidebarNavigation` 내부 두 분기 + `xl` 가시성으로 전환.** 컴포넌트가 절대 위치·고정 폭으로 짜여 CSS만으로 5rem 레일로 접히지 않으므로, 풀 분기(기존 마크업, `hidden xl:flex`)와 신규 아이콘 레일 분기(`xl:hidden`)를 한 컴포넌트에서 렌더해 스크립트·뮤테이션·스위처 상태를 공유한다. drawer surface는 항상 풀 분기를 쓴다. JS 미디어쿼리 없이 SSR-안전하다.
- **글쓰기 진입.** 아이콘 레일 단계엔 우측 컴포저·하단 탭이 없어, `BottomTabBar`의 `/compose` 항목을 사이드바 양 분기에 글쓰기 버튼으로 재사용한다.
- **프로필 스위처 드롭다운**은 snippet으로 추출해 풀(아바타 아래)·아이콘 레일(아바타 우측 popover) 양쪽에서 위치만 바꿔 재사용한다.

## Risks / Trade-offs

- 데스크톱 surface에서 두 분기가 모두 DOM에 존재한다(한쪽은 `display:none`). 폼 `id` 충돌을 막으려 snippet에 surface별 suffix를 둔다.
- 기존 풀 분기의 raw hex 색상은 이번 범위에서 유지한다(후속 토큰 마이그레이션). 신규 마크업만 디자인 토큰을 쓴다.
