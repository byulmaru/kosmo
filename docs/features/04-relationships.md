# 관계와 소셜 그래프

## 목표

사용자가 관심 있는 프로필을 팔로우하고, 팔로우 관계를 관리하고, 관계 기반 피드와 권한을
예측 가능하게 사용할 수 있어야 한다.

## 핵심 기능

### 팔로우

- 사용자는 다른 프로필을 팔로우할 수 있다.
- 팔로우한 프로필의 게시는 홈 타임라인에 나타난다.
- 자기 자신은 팔로우할 수 없다.
- 차단한 프로필 또는 나를 차단한 프로필은 팔로우할 수 없다.
- 원격 프로필 팔로우는 연합 delivery, 승인, 실패 재시도 상태를 가질 수 있다.

### 언팔로우

- 사용자는 팔로우를 해제할 수 있다.
- 언팔로우 후 해당 프로필의 새 게시는 홈 타임라인에 들어오지 않는다.
- 이미 받은 알림과 과거 타임라인 항목을 즉시 삭제할지 여부는 정책으로 정한다.
- 원격 언팔로우 실패 시 로컬 상태와 원격 상태 불일치 처리가 필요하다.

### 팔로우 요청

Mastodon 스타일을 우선 기준으로 둔다.

- 승인제 프로필을 팔로우하면 즉시 팔로우가 아니라 요청 상태가 된다.
- 대상 사용자는 요청을 승인하거나 거절할 수 있다.
- 요청자는 대기 상태를 볼 수 있고 취소할 수 있다.
- 승인 전에는 팔로워 공개 게시를 볼 수 없다.
- 원격 팔로우 요청은 ActivityPub follow/accept/reject 흐름과 연결된다.

### 팔로워 목록

- 프로필을 팔로우하는 프로필 목록을 보여준다.
- 목록 공개 범위는 프로필 공개 정책을 따른다.
- 목록은 cursor pagination을 제공한다.
- 각 항목에는 avatar, display name, display handle, bio 일부, follow action을 보여줄 수 있다.
- active profile 전환 시 follow button 상태가 현재 profile 기준으로 재계산되어야 한다.

### 팔로잉 목록

- 프로필이 팔로우하는 프로필 목록을 보여준다.
- 목록 공개 범위는 팔로워 목록과 별도로 정할 수 있다.
- cursor pagination을 제공한다.
- remote profile의 목록은 서버가 가져온 범위와 최신성에 제한이 있을 수 있다.

### 리스트

X, Mastodon의 list를 참고한다.

- 사용자는 프로필 묶음을 만들 수 있다.
- 리스트 타임라인에서 해당 프로필의 게시를 볼 수 있다.
- 리스트는 공개/비공개 설정을 가질 수 있다.
- 리스트에 넣은 사람이 알림을 받는지 정해야 한다.
- 팔로우하지 않은 프로필도 리스트에 넣을 수 있는지 정해야 한다.

### 추천 팔로우

- 초기 온보딩과 빈 홈 타임라인에서 추천 프로필을 보여줄 수 있다.
- 추천 사유를 보여주는 편이 좋다.
- 추천에서 숨기기, 관심 없음, 차단/뮤트가 가능해야 한다.
- 추천 알고리즘을 포함할지 제품 결정이 필요하며, 운영 curated 목록부터 시작할 수 있다.

## 참고 기능

### Starter Pack

Bluesky의 starter pack을 참고한다.

- 특정 주제나 커뮤니티에 필요한 프로필과 피드를 묶어 온보딩에 제공한다.
- 사용자는 한 번에 여러 프로필을 팔로우할 수 있다.
- 공식 starter pack과 사용자 생성 starter pack을 분리할 수 있다.
- Kosmo에서 온보딩/발견 기능으로 포함할지 제품 결정이 필요하다.

### 팔로우 가져오기와 내보내기

- Mastodon 계열의 CSV import/export를 참고할 수 있다.
- 계정 이동, 서버 이전, 백업과 연결된다.
- 원격 프로필 identity 안정성이 먼저 필요하다.

### 가까운 친구 또는 서클

- X의 일부 과거 기능이나 Instagram 계열 기능과 유사하다.
- 단문형 SNS 공통 핵심으로 볼지 제품 결정이 필요하다.
- 공개 범위가 복잡해지므로 팔로워 공개 이후 별도 검토가 필요하다.

## 상태와 에러

- follow button은 팔로우 안 함, 팔로잉, 요청 대기, 나 자신, 차단됨, 로딩, 실패 상태를 가진다.
- 요청 승인/거절 실패 시 목록 상태를 보존하고 재시도할 수 있어야 한다.
- 원격 follow delivery 실패는 사용자에게 즉시 실패로 보일 수도 있고 대기 상태로 보일 수도 있다.
- 관계 변경은 홈 타임라인, 프로필 통계, 알림, 추천에 영향을 준다.

## 데이터/정책 메모

- 팔로우 관계는 viewer profile 기준이다. 로그인 계정 기준과 혼동하면 안 된다.
- 계정이 여러 프로필을 가질 수 있으므로 active profile 전환 시 모든 관계 UI가 갱신되어야 한다.
- follow request와 follow relationship은 별도 상태로 모델링하는 편이 안전하다.
- block은 follow보다 우선한다. 차단 시 기존 follow를 자동 해제할지 정책이 필요하다.

## 미결정 네이밍

- 팔로우: Follow, Subscribe
- 팔로워: Followers
- 팔로잉: Following
- 팔로우 요청: Follow Request, Pending Follow
- 리스트: List, Circle, Collection
- 스타터팩: Starter Pack, Follow Pack, Onboarding Pack
