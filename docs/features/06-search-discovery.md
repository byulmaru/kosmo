# 검색과 발견

## 목표

사용자가 사람, 게시, 해시태그, 주제, 커뮤니티를 찾을 수 있어야 한다. 검색은 사용자가
명시적으로 찾는 기능이고, 발견은 사용자가 아직 모르는 대상을 만나는 기능이다.

## 핵심 기능

### 통합 검색

- 사용자는 검색어 하나로 게시, 프로필, 해시태그를 찾을 수 있다.
- 검색 결과는 탭으로 나누거나 섹션으로 나눌 수 있다.
- 검색어는 최근 검색으로 저장할 수 있다.
- 차단, 뮤트, 필터, 공개 범위 제한을 적용한다.
- 검색 색인에서 삭제/정지/비공개 전환을 반영해야 한다.

### 게시 검색

- 공개 게시를 검색할 수 있다.
- 팔로워 공개와 멘션 대상 공개 게시를 검색 결과에 노출하지 않는다.
- 작성자, 날짜, 해시태그, 미디어 포함 여부 같은 필터를 둘 수 있다.
- 원격 게시 검색은 서버가 색인한 범위에 제한된다.
- 전문 검색과 해시태그 검색은 다른 색인 정책을 가질 수 있다.

### 프로필 검색

- display name, handle, bio 일부로 프로필을 찾을 수 있다.
- 로컬 handle exact match를 우선 제공한다.
- 원격 actor lookup은 검색과 구분해서 명확히 보여준다.
- 같은 handle이 여러 instance에 있을 수 있으므로 host를 함께 표시한다.
- 정지/삭제/차단된 프로필은 검색에서 제외하거나 제한 표시한다.

### 해시태그 검색

- 사용자는 해시태그를 검색하고 해당 해시태그 타임라인으로 이동할 수 있다.
- 해시태그 자동완성과 인기 해시태그를 제공할 수 있다.
- 해시태그는 대소문자와 정규화 정책이 필요하다.
- spam, abuse, 성인/민감 태그 노출 정책이 필요하다.

### URL 또는 actor lookup

- 사용자는 원격 프로필 URL, ActivityPub actor URL, `acct:handle@host` 형태로 사람을 찾을 수
  있다.
- WebFinger lookup이 가능한 경우 로컬 actor URI로 resolve한다.
- 원격 fetch 실패, 없는 actor, 차단된 host를 명확히 구분한다.
- 검색 결과와 lookup 결과를 혼동하지 않도록 UI 문구를 나눈다.

## 발견 기능

### 탐색 탭

- 인기 게시, 추천 프로필, 인기 해시태그, 공식 공지, 추천 피드를 모아 보여줄 수 있다.
- 제품 결정에 따라 운영 curated 섹션부터 시작할 수 있다.
- 알고리즘 추천은 나중에 도입하더라도 숨기기/관심 없음/신고 흐름이 필요하다.

### 트렌드

X와 Mastodon의 trending tag/post 계열을 참고한다.

- 특정 기간에 많이 언급된 해시태그 또는 게시를 보여준다.
- 트렌드는 spam과 brigading에 취약하므로 moderation queue가 필요하다.
- 서버 단위 trend와 전체 네트워크 trend를 구분해야 한다.
- trend 포함 여부와 운영자 승인 모델은 제품 결정이 필요하다.

### 커스텀 피드 발견

Bluesky custom feed를 참고한다.

- 사용자는 topic feed를 검색하고 고정할 수 있다.
- feed creator, 설명, 샘플 게시, 구독자 수를 보여줄 수 있다.
- 외부 feed generator를 허용하면 신뢰와 개인정보 정책이 필요하다.
- 공식 feed만 제공하는 방식은 단순한 선택지다.

### 스타터팩 발견

Bluesky starter pack을 참고한다.

- 신규 사용자가 관심사별 추천 프로필/피드를 한 번에 따라갈 수 있다.
- starter pack은 주제, 설명, 포함 프로필, 포함 피드를 가진다.
- 사용자는 전체 팔로우 또는 일부 선택을 할 수 있다.
- 운영 curated starter pack과 사용자 생성 starter pack의 포함 범위는 제품 결정이 필요하다.

### 안테나 발견

Misskey antenna를 참고한다.

- 사용자는 키워드 조건을 저장하고 새 게시를 지속적으로 모을 수 있다.
- 검색보다 능동적인 구독형 발견 기능이다.
- 조건이 너무 넓으면 알림/저장 비용이 커지므로 제한이 필요하다.
- 고급 사용자 기능으로 볼지 제품 결정이 필요하다.

## 검색 권한과 안전

- 차단한 프로필과 나를 차단한 프로필은 검색 결과에서 제한한다.
- 뮤트한 단어가 포함된 게시는 검색에서도 숨기거나 접는다.
- 민감 미디어는 검색 결과에서 자동 펼침하지 않는다.
- 신고/정지된 콘텐츠는 색인에서 제외한다.
- 서버 차단 도메인의 원격 콘텐츠는 lookup도 제한할 수 있다.

## 상태와 에러

- 검색어 없음: 최근 검색, 추천 검색어, 인기 해시태그.
- 결과 없음: spelling suggestion, 원격 lookup 제안.
- 검색 실패: 네트워크 오류와 서버 오류를 분리.
- 원격 lookup 실패: host unreachable, not found, blocked, unsupported를 구분.
- pagination 실패: 기존 결과를 유지하고 재시도.

## 데이터/정책 메모

- 검색 색인은 제품 데이터와 다른 수명주기를 가진다.
- 삭제/비공개/차단 상태 변경이 색인에 반영되는 지연을 줄여야 한다.
- 원격 콘텐츠 색인은 서버가 실제로 관측한 범위만 다룬다.
- 검색 로그와 최근 검색은 개인정보 정책이 필요하다.
