# 타임라인과 피드

## 목표

사용자가 팔로우한 사람, 같은 서버의 사람, 연합 네트워크, 주제 기반 피드에서 게시를 탐색할 수
있어야 한다. 기본 모델은 Mastodon의 홈/로컬/연합 타임라인을 우선 기준으로 두고, Bluesky의
custom feed와 Misskey의 antenna/channel은 확장 참고로 둔다.

## 핵심 타임라인

### 홈 타임라인

- 사용자가 팔로우한 프로필의 게시를 최신순으로 보여준다.
- 답글 포함 여부는 정책으로 정한다.
- 부스트/리포스트/리노트는 팔로우 관계를 통해 홈 타임라인에 들어올 수 있다.
- 뮤트, 차단, 필터, 공개 범위 제한을 반영해야 한다.
- 새 게시가 있을 때 자동 삽입할지, 사용자가 버튼으로 불러오게 할지 정해야 한다.
- 커서 기반 pagination을 제공한다.

### 로컬 타임라인

Mastodon 스타일을 우선 기준으로 둔다.

- 같은 Kosmo 서버 또는 같은 인스턴스의 공개 게시를 보여준다.
- 팔로우하지 않은 로컬 사용자를 발견하는 기능이다.
- 팔로워 공개, 멘션 대상 공개, 검색 제외 공개는 로컬 타임라인에 들어가지 않는다.
- 서버 정책으로 로컬 타임라인 노출 제외 계정 또는 게시를 둘 수 있어야 한다.

### 연합 타임라인

Mastodon 스타일을 우선 기준으로 둔다.

- 서버가 알고 있는 원격 actor의 공개 게시를 보여준다.
- 원격 전체 네트워크의 완전한 목록이 아니라 서버가 관측한 범위의 피드라는 점을 UI 문구로
  명확히 해야 한다.
- 도메인 차단, 서버 차단, 원격 삭제, 원격 계정 정지를 반영해야 한다.
- 제품 결정에 따라 연합 타임라인을 숨기고 내부 수집 모델만 준비할 수 있다.

### 프로필 게시 목록

- 특정 프로필이 작성한 게시 목록을 보여준다.
- 고정 게시, 답글 포함 여부, 미디어만 보기 같은 필터를 제공할 수 있다.
- 방문자와 작성자의 관계에 따라 보이는 게시가 달라진다.
- 로컬 프로필과 원격 프로필 표시 정책을 분리해야 한다.

### 해시태그 타임라인

- 특정 해시태그가 포함된 공개 게시 목록을 보여준다.
- 팔로워 공개와 멘션 대상 공개 게시는 제외한다.
- 뮤트/차단/필터를 반영한다.
- 해시태그 대소문자, 정규화, 언어별 분절 정책을 정해야 한다.

## 참고 피드

### 커스텀 피드

Bluesky의 custom feed를 참고한다.

- 사용자는 특정 기준으로 생성된 피드를 구독하거나 고정할 수 있다.
- 피드는 공식 피드, 커뮤니티 피드, 외부 알고리즘 피드로 나뉠 수 있다.
- 제품 결정에 따라 사용자가 직접 피드 알고리즘을 만들게 하지 않고 운영자가 제공하는 피드부터 둘 수
  있다.
- 피드 결과는 왜 이 게시가 나타났는지 설명 가능해야 한다.

### 리스트 피드

X와 Mastodon의 list를 참고한다.

- 사용자는 프로필 묶음을 만들고 해당 프로필 게시만 모아 볼 수 있다.
- 리스트는 공개/비공개를 가질 수 있다.
- 팔로우하지 않은 프로필을 리스트에 넣을 수 있는지 정해야 한다.
- 리스트 타임라인은 홈 타임라인과 별도 읽음 상태를 가질 수 있다.

### 안테나

Misskey의 antenna를 참고한다.

- 키워드, 사용자, 출처, 공개 범위 조건으로 게시를 수집한다.
- 검색보다 지속적인 수집에 가깝다.
- 사용자는 안테나 조건을 저장하고 나중에 다시 볼 수 있다.
- 고급 기능으로 볼지, 검색/해시태그/리스트와 함께 다룰지 제품 결정이 필요하다.

### 채널 또는 커뮤니티 피드

Misskey channel과 X Communities를 참고한다.

- 특정 주제 또는 그룹 안에서 게시를 모아 본다.
- 일반 공개 타임라인과 독립된 규칙, 관리자, 멤버십을 가질 수 있다.
- Kosmo에서는 커뮤니티 기능을 별도 제품 도메인으로 둘지 먼저 결정해야 한다.

## 정렬과 추천

- 기본 정렬은 최신순이다.
- 추천/알고리즘 정렬을 포함할지 제품 결정이 필요하다.
- 추천을 도입한다면 사용자가 최신순으로 되돌릴 수 있어야 한다.
- 추천 사유, 숨기기, 관심 없음, 계정/주제 뮤트가 함께 필요하다.
- Bluesky처럼 사용자가 피드를 선택하는 모델은 중앙 추천보다 Kosmo 방향에 더 잘 맞을 수 있다.

## 타임라인 필터링

모든 타임라인은 다음 필터를 공통으로 적용해야 한다.

- 내가 차단한 프로필의 게시 제외.
- 나를 차단한 프로필의 비공개 게시 제외.
- 내가 뮤트한 프로필의 게시 숨김 또는 접힘.
- 내가 설정한 단어/태그 필터 적용.
- 서버가 차단한 계정, 도메인, 미디어 제외.
- 삭제, 정지, tombstone 상태 반영.

## 상태와 에러

- 최초 로딩: skeleton 또는 빈 상태.
- 빈 타임라인: 팔로우 추천, 검색, 로컬 타임라인 진입을 제안.
- 추가 로드 실패: 현재 목록은 보존하고 재시도 버튼 제공.
- 새 게시 있음: 자동 스크롤하지 않고 새 게시 로드 버튼 제공.
- 필터로 모두 숨겨진 상태: 숨김 사유를 노출할지 정책으로 정한다.

## 데이터/정책 메모

- 타임라인은 게시 원본과 타임라인 삽입 이벤트를 분리해서 생각해야 한다.
- 부스트/리포스트는 원본 게시 재사용과 타임라인 이벤트를 함께 가진다.
- 커서 pagination은 삭제/삽입에 안정적이어야 한다.
- 연합 타임라인은 수집 범위를 사용자에게 과장해서 보여주면 안 된다.

## 미결정 네이밍

- 타임라인: Timeline, Feed
- 홈: Home, Following
- 로컬: Local, Server
- 연합: Federated, Network
- 커스텀 피드: Custom Feed, Saved Feed, Topic Feed
- 안테나: Antenna, Watch, Keyword Feed
