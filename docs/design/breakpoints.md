# 레이아웃 브레이크포인트

KOSMO 웹의 메인 3분할 레이아웃은 트위터/X처럼 화면 폭에 따라 단계적으로 컬럼이 줄어든다. 별도 커스텀 브레이크포인트 토큰을 만들지 않고 **Tailwind 기본 브레이크포인트(`md`/`lg`)를 그대로 재사용**한다. 좁은 데스크톱 폭에서 일찍 압축되도록 풀 3분할 등장 폭을 `lg`(1024px)로 둔다.

## 단계

| 단계            | 폭 구간              | 좌측        | 중앙         | 우측   | 모바일 셸            |
| --------------- | -------------------- | ----------- | ------------ | ------ | -------------------- |
| 1 모바일        | `< md` (768px 미만)  | 드로어      | 피드(전체폭) | —      | ☰ 헤더 + 하단 탭 바 |
| 2 아이콘 + 피드 | `md`~`lg` (768~1024) | 아이콘 레일 | 피드         | —      | 없음                 |
| 3 풀 3분할      | `≥ lg` (1024px 이상) | 풀 사이드바 | 피드         | 컴포저 | 없음                 |

- **`md`(768px)** = 모바일 ↔ 데스크톱 경계. 미만은 하단 탭 바 + 드로어 사이드바, 이상은 사이드바가 항상 보인다.
- **`lg`(1024px)** = 좌측 풀 사이드바(프로필 헤더 + 라벨)와 우측 컴포저(우측 레일)가 함께 등장해 풀 3분할이 된다. `md`~`lg`는 좌측이 아이콘 전용 레일이고 우측 레일이 없다.

각 컬럼 폭(풀 사이드바 `20rem` / 아이콘 레일 `5rem`, 중앙 `minmax(0,600px)`, 우측 `minmax(290px,350px)`)을 더하면 위 경계에서 레이아웃이 눌리지 않고 들어맞는다. 중앙은 `lg` 시작점에서 600px보다 좁게(≈414px) 수축됐다가 폭이 늘면 600까지 자란다.

## 글쓰기 진입

- `< md`: 하단 탭 바의 글쓰기.
- `md`~`lg`: 우측 레일이 없으므로 아이콘 레일의 글쓰기 버튼.
- `≥ lg`: 우측 레일 컴포저가 담당하며, 사이드바 글쓰기 버튼은 표시하지 않는다(드로어 surface에서만 유지).

## 구현 위치

- 셸 그리드와 컬럼 가시성: `apps/web/src/routes/(tabs)/+layout.svelte`
- 접힌/펼친 사이드바 분기(`lg` 기준 전환): `apps/web/src/lib/components/SidebarNavigation.svelte`
- 하단 탭 바 가시성(`md:hidden`): `apps/web/src/lib/components/BottomTabBar.svelte`

## 컨벤션 (다른 화면에서 재사용)

- 모바일 ↔ 데스크톱 셸 전환은 `md`를 기준으로 한다(`md:hidden`, `hidden md:block`).
- 우측 보조 컬럼(레일)과 풀 사이드바 등 데스크톱 전체 구성은 `lg`를 기준으로 노출한다.
- 커스텀 `--breakpoint-*` 토큰을 새로 만들지 않는다. 기본 `sm`/`md`/`lg`/`xl`/`2xl`로 표현되지 않는 단계가 꼭 필요할 때만 디자인 오너와 합의 후 추가한다.

## Figma 대응

`05 Screens - Web`의 프레임과 압축형 단계는 다음과 같이 대응한다. 압축형 재조정으로 일부 프레임이 코드 단계와 어긋나므로 Figma 프레임 정리는 후속 작업으로 남긴다.

- **1440 프레임**(풀 사이드바 + 피드 + 컴포저) = 3단계(`≥ lg`).
- **1024 프레임**(접힌 아이콘 메뉴 + 피드 + 컴포저)은 압축형에서 더 이상 단일 단계와 1:1로 맞지 않는다. 코드는 `lg`(1024px)부터 풀 사이드바로 전환하며 우측 컴포저를 포함하고, `md`~`lg` 2단계에는 우측 컴포저가 없다. 후속에서 2단계(아이콘 레일 + 피드)·3단계 기준으로 프레임을 정리한다.
