# 레이아웃 브레이크포인트

KOSMO 웹의 메인 3분할 레이아웃은 트위터/X처럼 화면 폭에 따라 단계적으로 컬럼이 줄어든다. 별도 커스텀 브레이크포인트 토큰을 만들지 않고 **Tailwind 기본 브레이크포인트(`md`/`lg`/`xl`)를 그대로 재사용**한다.

## 단계

| 단계            | 폭 구간               | 좌측        | 중앙         | 우측   | 모바일 셸            |
| --------------- | --------------------- | ----------- | ------------ | ------ | -------------------- |
| 1 모바일        | `< md` (768px 미만)   | 드로어      | 피드(전체폭) | —      | ☰ 헤더 + 하단 탭 바 |
| 2 아이콘 + 피드 | `md`~`lg` (768~1024)  | 아이콘 레일 | 피드         | —      | 없음                 |
| 3 + 글쓰기 바   | `lg`~`xl` (1024~1280) | 아이콘 레일 | 피드         | 컴포저 | 없음                 |
| 4 기본 3분할    | `≥ xl` (1280px 이상)  | 풀 사이드바 | 피드         | 컴포저 | 없음                 |

- **`md`(768px)** = 모바일 ↔ 데스크톱 경계. 미만은 하단 탭 바 + 드로어 사이드바, 이상은 (접힌) 사이드바가 항상 보인다.
- **`lg`(1024px)** = 우측 컴포저(글쓰기 바) 등장.
- **`xl`(1280px)** = 좌측 사이드바가 아이콘 레일에서 풀 사이드바(프로필 헤더 + 라벨)로 펼쳐진다.

각 컬럼 폭(풀 사이드바 `20rem` / 아이콘 레일 `5rem`, 중앙 `minmax(0,600px)`, 우측 `minmax(290px,350px)`)을 더하면 위 경계에서 레이아웃이 눌리지 않고 들어맞는다.

## 구현 위치

- 셸 그리드와 컬럼 가시성: `apps/web/src/routes/(tabs)/+layout.svelte`
- 접힌/펼친 사이드바 분기(`xl` 기준 전환): `apps/web/src/lib/components/SidebarNavigation.svelte`
- 하단 탭 바 가시성(`md:hidden`): `apps/web/src/lib/components/BottomTabBar.svelte`

## 컨벤션 (다른 화면에서 재사용)

- 모바일 ↔ 데스크톱 셸 전환은 `md`를 기준으로 한다(`md:hidden`, `hidden md:block`).
- 우측 보조 컬럼(레일)은 `lg`부터 노출한다.
- 와이드 데스크톱 전용 확장(사이드바 펼침 등)은 `xl`을 기준으로 한다.
- 커스텀 `--breakpoint-*` 토큰을 새로 만들지 않는다. 기본 `sm`/`md`/`lg`/`xl`/`2xl`로 표현되지 않는 단계가 꼭 필요할 때만 디자인 오너와 합의 후 추가한다.

## Figma 대응

`05 Screens - Web`의 프레임은 이 단계 중 일부에 대응한다.

- **1024 프레임** = 3단계(접힌 아이콘 메뉴 + 피드 + 컴포저)
- **1440 프레임** = 4단계(풀 사이드바 + 피드 + 컴포저)

모바일(1단계)·아이콘+피드(2단계)는 현재 별도 웹 프레임이 없어 코드에서만 정의한다.
